  <name>Alpaca bar fetching and technical indicator computation</name>
  <files>screener/market_data.py, tests/test_market_data.py</files>
  <behavior>
    fetch_daily_bars(client, symbols, num_bars=250, batch_size=20) -> dict[str, pd.DataFrame]:
    - Computes start date as now - (num_bars * 1.5) calendar days to cover ~num_bars trading days
    - Iterates symbols in batches of batch_size
    - Creates StockBarsRequest with TimeFrame.Day, Adjustment.SPLIT, no limit parameter
    - Extracts per-symbol DataFrames from multi-index response
    - Symbols missing from response are silently skipped (not in returned dict)
    - Returns dict mapping symbol -> DataFrame with columns including 'close' and 'volume'

    compute_indicators(bars_df: pd.DataFrame) -> dict:
    - Takes a single symbol's bar DataFrame (must have 'close' and 'volume' columns)
    - Returns dict with keys: 'price', 'avg_volume', 'rsi_14', 'sma_200', 'above_sma200'

    price: float(close.iloc[-1]) -- last closing price

    avg_volume: float(volume.mean()) -- average daily volume across all bars

    rsi_14:
    - If len(close) >= 30: compute RSIIndicator(close, window=14).rsi(), take iloc[-1]
    - If result is NaN (pd.isna): return None
    - If len(close) < 30: return None

    sma_200:
    - If len(close) >= 200: compute SMAIndicator(close, window=200).sma_indicator(), take iloc[-1]
    - If result is NaN: return None
    - If len(close) < 200: return None

    above_sma200:
    - If sma_200 is not None: return price > sma_200
    - Otherwise: return None
  </behavior>
