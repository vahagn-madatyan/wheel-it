<feature>
  <name>FinnhubClient with rate limiting, retry, and metric fallback chains</name>
  <files>screener/finnhub_client.py, tests/test_finnhub_client.py, pyproject.toml</files>
  <behavior>
    Rate Limiting (SAFE-02):
    - _throttle() sleeps to maintain at least call_interval seconds between API calls
    - Two consecutive calls made less than call_interval apart result in a sleep of (call_interval - elapsed)
    - Default call_interval is 1.1 seconds (~54 calls/min, safely under 60/min)

    429 Retry (SAFE-02):
    - On first 429 FinnhubAPIException: log WARNING, sleep 5s, retry once
    - On second 429 (retry also fails): let exception propagate (caller handles skip)
    - Non-429 FinnhubAPIException: re-raise immediately (no retry)

    company_profile(symbol) -> dict:
    - Calls finnhub.Client.company_profile2(symbol=symbol) with throttle
    - Returns dict with marketCapitalization, finnhubIndustry, ticker, etc.
    - Empty response (symbol not found): returns empty dict {}

    company_metrics(symbol) -> dict:
    - Calls finnhub.Client.company_basic_financials(symbol, "all") with throttle
    - Returns the response dict (has 'metric' key with sub-dict)
    - Empty response: returns dict with empty metric sub-dict

    extract_metric(metrics, chain_name) -> float | None (SAFE-04):
    - Given a metrics dict and a chain name ("debt_equity", "net_margin", "sales_growth")
    - Tries each key in METRIC_FALLBACK_CHAINS[chain_name] in order
    - Returns first non-None value as float
    - Returns None if all keys missing/null

    METRIC_FALLBACK_CHAINS constant (SAFE-04):
    - "debt_equity": ["totalDebtToEquity", "totalDebtToEquityQuarterly", "totalDebtToEquityAnnual"]
    - "net_margin": ["netProfitMarginTTM", "netProfitMarginAnnual", "netMargin"]
    - "sales_growth": ["revenueGrowthQuarterlyYoy", "revenueGrowth5Y", "revenueGrowth3Y"]

    Debug logging:
    - Each API call logs at DEBUG level: symbol, endpoint name, response time in seconds
  </behavior>
