  <files>tests/test_credentials.py</files>
  <action>
Fix the two failing tests (`test_finnhub_key_loaded` and `test_finnhub_key_missing_is_none`) by patching `dotenv.load_dotenv` to a no-op BEFORE calling `importlib.reload(creds)`.

Root cause: `load_dotenv(override=True)` at module level in `config/credentials.py` runs during `importlib.reload()`, reading the real `.env` file and overwriting monkeypatched env vars.

For BOTH `test_finnhub_key_loaded` and `test_finnhub_key_missing_is_none`, add this line before `importlib.reload(creds)`:

```python
monkeypatch.setattr("dotenv.load_dotenv", lambda *a, **kw: None)
```

This patches `dotenv.load_dotenv` at the source module, so when `credentials.py` runs `from dotenv import load_dotenv` during reload, it picks up the no-op. Patch at `dotenv.load_dotenv` (not `config.credentials.load_dotenv`) because reload creates a fresh binding from the source module.

The other two tests (`test_require_finnhub_key_returns_key` and `test_require_finnhub_key_raises_when_missing`) already pass and must NOT be changed.
